解析库

解析器 使用方法 优势 劣势
Python 标准库 BeautifulSoup(markup, ‘html.parser’) Python 的内置标准库、执行速度适中，文档容错能力强 旧版本 Python 中文容错能力差
lxml HTML 解析库 BeautifulSoup(markup, ‘lxml’) 速度快，容错能力强 需要安装 C 语言库
lxml XML 解析库 BeautifulSoup(markup, ‘lxml’) 速度快，唯一支持 XML 的解析库 需要安装 C 语言库
html5lib BeautifulSoup(markup, ‘html5lib’) 最好的容错性，以浏览器的方式解析文档，生成 html5 格式的文档 速度慢

基本使用

标签选择器

选择元素

soup.title
soup.head
soup.p

获取名称

soup.title.name

获取属性

soup.p.attrs[‘name’]
soup.p[‘name’]

获取内容

soup.p.string

嵌套选择

soup.head.title.string

子节点和孙节点

soup.p.contents
soup.p.children（返回可迭代对象）
soup.p.descendants（子孙节点，可迭代对象）

父节点和祖先节点

soup.a.parent
soup.a.parents（祖先节点，返回对迭代对象）

兄弟节点

soup.a.next_siblings
soup.a.previous_siblings

标准选择器

find_all(name,attrs,recursive,text,**kwargs)

标签名

soup.find_all(‘ul’)（返回标签列表）

属性

soup.find_all(attrs={‘id’:’list-1’})
soup.find_all(id=’list-1’)
soup.findall(class=’element’) （class 为 Python 的关键字）

内容

soup.find_all(text=’Foo’) （返回内容）

find(name,attrs,recursive,text,**kwargs)

与 find_all 类似，返回第一个匹配元素

find_parents()
find_parent()

find_next_siblings()
find_next_sibling()

find_previous_siblings()
find_previous_sibling()

find_all_next()
find_next()

find_all_previous()
find_previous()

CSS 选择器

通过 select() 直接传入选择器

soup.select(‘ul li’)
soup.select(‘#list-2 .element’)
返回元素列表

获取属性

ul[‘id’]
ul.attrs[‘id’]

获取内容

ul.get_text()

总结

推荐使用 lxml 库解析，必要时使用html.parser
标签选择器筛选功能弱但是速度快
使用 find()、find_all() 查询匹配单个或者多个结果
如果对 CSS 选择器熟悉使用 select()
记住常用的获取属性和文本值的方法
