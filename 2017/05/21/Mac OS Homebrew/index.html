常用指令

安装软件包
brew install <package_name>

重新安装软件包（没有重新下载，而是用 /Library/Caches/Homebrew 的缓存文件重新安装）
brew reinstall <package_name>

卸载软件包
brew uninstall <package_name>

卸载并删除软件包
brew remove <package_name>

搜索软件包
brew search <package_name>

更新 Homebrew 在服务器端上的包目录
brew update

查询哪些包需要更新
brew outdated

更新包
brew upgrade <package_name>

清理旧版本包的缓存
brew cleanup

查看安装过的包列表
brew list --versions

查看软件包的信息
brew info <package_name>

查看软件包的依赖
brew deps <package_name>

brew 命令预演参数，如：
brew cleanup -n
预演命令，而不是真正的执行

管理服务
brew services list

重启 redis 服务(start、stop 同理）
brew services restart redis

安装路径

Homebrew 将本机的/usr/local/目录初始化为 Git 的工作树，并将目录所有者变更为 $USER，也就是当前所操作的用户，所以以后的操作都不再需要sudo，这是安全的，全新的OS X默认是没有该目录的，也就是说该目录并非是系统所有。

brew 大多是直接下载源码然后编译安装，安装命令如 ./configure && make install。并且，在安装本软件之前，会先安装脚本里指明的本项目的依赖库。

软件安装的目录是 /usr/local/Cellar/<package_name>/<version>/。源码编译完成后，生成的文件都会放在这里，如果是直接下载的可执行文件，也会直接放在这里。每个版本的软件都会放在对应版本名称的文件夹下。一般的软件可能会包含可执行文件、供其它库使用的头文件、运行库、文档等文件或目录，为了方便升级管理，，brew 会把这些文件或目录分别软链接到 /usr/local/bin/、/usr/local/include/、/usr/local/lib/、/usr/local/share/ 等目录下。

如果你用 brew upgrade 更新过几次该软件，你所有安装过的版本都会在 /usr/local/Cellar// 目录下找到。每次升级时，brew都是先把可执行文件等都编译到当前版本目录下，撤销之前版本的软链接，然后再将新版本软链接到 /usr/local 的各子文件夹下。这样，我们就会使用到最新版本了。
