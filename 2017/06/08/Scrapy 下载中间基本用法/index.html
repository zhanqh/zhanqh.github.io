处理 scrapy 框架 request/response 的钩子

开启下载中间件&禁用某中间件

process_request(request, spider)

  • 返回 None ：继续调用下一个 process_request() 方法或者发起请求
  • 返回 Response ：不再调用 process_request() 方法与 process_exception() 方法，接下去调用 process_response() 方法或者直接返回 response
  • 返回 Request ：将 request 推入到 scheduler 中，不调用其他函数
  • raise IgnoreRequest ：调用 process_exception() 方法

process_response(request, response, spider)

  • 返回 Response ：不再调用 process_request() 方法与 process_exception() 方法，接下去调用 process_response() 方法或者直接返回 response
  • 返回 Request ：将 request 推入到 scheduler 中，不调用其他函数
  • raise IgnoreRequest ：调用 process_exception() 方法

process_exception(request, exception, spider)

  • 返回 None ：继续调用下一个 process_exception() 方法
  • 返回 Response ：不再调用 process_request() 方法与 process_exception() 方法，接下去调用 process_response() 方法或者直接返回 response
  • 返回 Request ：将 request 推入到 scheduler 中，不调用其他函数

获取项目 DOWNLOADER_MIDDLEWARES 信息

scrapy settings --get DOWNLOADER_MIDDLEWARES_BASE
