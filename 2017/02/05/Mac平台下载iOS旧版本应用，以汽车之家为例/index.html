前言

iOS 平台上 app 应用只有唯一的官方安装渠道，用户要么不更新应用，要么只能更新至最新的版本，不同于安卓有许多的应用安装途径，可以随意选择版本。在中国平安入主汽车之家之后，对汽车之家的移动端应用不但更改了UI逻辑，某些业务逻辑是相当的傻逼，更糟糕的是还进行了近乎竭泽而渔的商业化，原本与汽车相关的广告出现频率高点还能接受，现在是什么乱七八糟的广告都有，榨干汽车之家移动端流量的最后一点价值，完全不顾用户体验了。7.0之后的应用实在是没法用了，Google 搜寻了一番没有找能用的汽车之家旧版本 ipa 安装包，但是找到了这篇教程：【技术贴】IOS如何下载旧版本应用APP，Windows 平台下利用 fiddler 拦截、构造请求下载旧版本应用。仿照此教程在 Mac 平台下利用 Charles 亦实现了 iOS 旧版本应用的下载。转载需保留原教程出处。

实现原理

通过抓包软件查找 App 的历史版本 ID，然后通过拦截、构造 iTunes 的下载请求，将请求中最新的版本 ID 改成历史版本 ID，以下载指定版本的 App。

Charles 配置

设置 Charles 代理 Mac OS 的全部网络请求

iTunes 下载应用是 https 请求，Charles 必须安装证书后才能抓取 https 请求。下载 SSL 证书

直接添加证书会出现不能修改“System Roots”钥匙串的错误，解决方法：

  • 打开钥匙串访问 ➞ 登录，然后将证书拖进去
  • 双击证书，修改权限
  • 开启 SSL 代理功能，参考教程

用 Charles 抓取 iTunes 的 https 请求，获取 App 历史版本号

  • iTunes 中搜索汽车之家，并下载
  • Charles 中查看p26-buy.itunes.apple.com请求，两位数字不固定，path 路径为：/WebObjects/MZBuy.woa/wa/buyProduct，用过滤器buy.筛选出请求
  • 查看 App Store 中应用的版本记录，倒推出 6.2.1 版本的 ID 应该为 818664328

    使用 Charles 拦截并修改 https 请求

  • 通过 Charles 的 breakpoints 选项设置截断 iTunes 的 https 请求，参考教程
  • 再 iTunes 中再次点击下载汽车之家，此时 iTunes 的请求会被截断，Charles 会弹出如下界面，点击 Edit Request，将最新的应用版本号改成 818664328，然后点击 Execute 继续即可。
  • 下载完成后在 iTunes 的资料库中用 Finder 打开汽车之家安装包，成功下载到 6.2.1 版本的汽车之家，使用同步推安装到 iPhone 即可。
